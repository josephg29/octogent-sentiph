The Problem

The Fix

Octoboss delegates. You just think.

Octoboss gets its own AI session — a permanent, always-on brain that lives at the center of your canvas. But unlike regular sessions, it can reach out and spin up new worker sessions on its own, send them tasks, and read their results back — all without you lifting a finger.

What You Get

Four new superpowers.

Everything else stays exactly the same. These are the only additions.

🧠

A permanent boss session

Octoboss always has its own Claude Code terminal ready. Open it from the canvas at any time — it's a regular Claude Code session, nothing special about it until you need to delegate.

🚀

Spawn workers on command

Tell Octoboss to delegate a task. It creates a new terminal, types the prompt in, and starts the work — without you touching anything.

📬

Read results back

When a worker finishes, Octoboss can check in on it and pull the output back into the conversation — so you get a summary without switching tabs.

🎨

See it all on the canvas

Every terminal Octoboss spawns appears as a node on the canvas in real time. You can see what's running, what's done, and what Octoboss is thinking — at a glance.

How We Build It

Four steps. Smallest possible change.

We're not rebuilding anything. We're adding a thin layer on top of what already works.

1

Give Octoboss a terminal

The Octoboss node on the canvas gets a persistent Claude Code session that auto-creates on first click. It looks and works exactly like any other terminal.

2

Add a private tool server

A tiny private server starts alongside the app. It knows about all running terminals and exposes two actions: "spawn a terminal with this prompt" and "fetch this terminal's output."

3

Connect Octoboss to the tool server

Octoboss's Claude Code session is started with access to those two tools. From Octoboss's perspective, delegating feels like using any other built-in tool — it just works.

4

Light up the canvas

Terminals that Octoboss spawns are visually linked to it on the canvas. You can see the delegation graph in real time — who spawned what, what's running, what's done.
